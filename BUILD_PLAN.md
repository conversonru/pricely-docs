# Pricely — Step-by-Step Build Plan для Claude Code

## 🎯 Твоя задача
Собрать MVP сервиса Pricely за 7 дней, используя no-code/low-code инструменты (Softr, Glide, Make.com).

## 📅 План по дням

### День 1: Настройка базы данных (Google Sheets)
1. Создать Google Sheets с 3 таблицами: `Products`, `Clients`, `Orders`.
2. Заполнить `Products` тестовыми данными (20–30 товаров).
3. Настроить права доступа: read-only для Softr/Glide.

**Результат:** Готовая база данных.

### День 2: Сборка SEO-сайта на Softr
1. Зарегистрироваться на Softr.io, выбрать шаблон «Catalog».
2. Подключить Google Sheets (`Products`) как источник данных.
3. Настроить отображение:
   - Список товаров (карточки: фото, название, цена).
   - Страница товара (H1, описание, кнопка «Заказать»).
4. Настроить фильтры (по категории, цене).
5. Включить SEO:
   - Чистые URL (`/tovar-{Slug}`).
   - Шаблоны Title/Description (из столбцов `SEO_Title`, `SEO_Description`).
6. Подключить кастомный домен (тестовый: `demo.pricely.ru`).

**Результат:** Работающий SEO-сайт.

### День 3: Сборка Telegram Mini App на Glide
1. Зарегистрироваться на Glideapps.com, создать новый проект.
2. Подключить ту же Google Sheets (`Products`).
3. Выбрать шаблон «Store» или «Catalog».
4. Настроить кнопку «Заказать»:
   - Действие: «Open Link».
   - URL: `https://wa.me/{CLIENT_WHATSAPP}?text=Здравствуйте, хочу заказать: {Name}`.
5. В настройках включить «PWA» (Progressive Web App).
6. Протестировать на мобильном.

**Результат:** Работающий Mini App (пока без Telegram).

### День 4: Интеграция с Telegram
1. Создать бота в @BotFather (`/newbot`).
2. В настройках бота добавить кнопку меню «🛍 Каталог» → URL Glide-приложения.
3. Настроить команду `/start` (приветствие + кнопка).
4. Протестировать: открыть бота → нажать «Каталог» → Mini App должен открыться внутри Telegram.

**Результат:** Полноценный Telegram Mini App.

### День 5: Автоматизация загрузки Excel (Make.com)
1. Зарегистрироваться в Make.com.
2. Создать сценарий:
   - Триггер: Новый файл в Google Drive (папка `/pricely-uploads`).
   - Действие 1: XLSX Parser (разобрать файл).
   - Действие 2: Google Sheets (очистить `Products`, записать новые строки).
   - Действие 3: Telegram (уведомить клиента: «Прайс обновлён!»).
3. Протестировать: загрузить тестовый Excel → проверить, что данные появились в Sheets.

**Результат:** Авто-загрузка прайсов.

### День 6: Подключение платежей (Prodamus)
1. Зарегистрироваться в Prodamus (как самозанятый).
2. Создать 3 товара (подписки): Telegram, Business, PRO.
3. Настроить webhook на Make.com (успешная оплата → обновление `Clients`).
4. Добавить ссылки на оплату в Softr/Glide (или на лендинг).

**Результат:** Приём платежей.

### День 7: Финальное тестирование и запуск
1. Протестировать полный цикл:
   - Загрузить Excel → проверить сайт и Mini App → сделать тестовый заказ → проверить уведомление.
2. Исправить баги.
3. Опубликовать лендинг (Tilda/Softr) с тарифами.
4. Написать 5–10 потенциальным клиентам (тестовый запуск).

**Результат:** Готовый MVP.

## 🛠 Инструменты для Claude Code
- **Google Sheets API:** Для чтения/записи данных.
- **Softr API:** Для авто-создания сайтов (опционально).
- **Glide API:** Для авто-создания Mini App (опционально).
- **Make.com Webhooks:** Для автоматизации.

## ✅ Критерии готовности
- [ ] Сайт загружается < 2 сек.
- [ ] Mini App открывается в Telegram без ошибок.
- [ ] Кнопка «Заказать» отправляет сообщение в WhatsApp.
- [ ] Excel загружается автоматически за < 1 мин.
- [ ] Платежи проходят успешно.
